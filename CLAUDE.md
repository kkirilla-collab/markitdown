# CLAUDE.md — память проекта markitdown

Этот файл — постоянная память Claude для этого репозитория. Читается в начале каждой сессии.

## О проекте

`markitdown` — инструмент для конвертации файлов и офисных документов в Markdown.
Монорепо, пакеты в `packages/`:

| Пакет | Назначение |
|---|---|
| `markitdown` | Основная библиотека и CLI конвертации в Markdown |
| `markitdown-mcp` | MCP-сервер поверх markitdown |
| `markitdown-ocr` | OCR-расширение |
| `markitdown-sample-plugin` | Пример плагина (шаблон для расширений) |

Python-проект. Тесты/линтеры запускаются на уровне каждого пакета (см. `pyproject.toml` внутри пакетов).

## ПОСТОЯННОЕ ПРАВИЛО: тесты перед коммитом

> **Никогда не коммитить код без прогона тестов.** Это не опция — это часть задачи.
> Алгоритм для любого изменения кода:
> 1. Внести изменения
> 2. Запустить `pytest -k "not remote and not speech"` в затронутых пакетах
> 3. Убедиться, что новых падений нет (pre-existing баги — фиксировать явно, не замалчивать)
> 4. Только после этого коммитить и пушить
>
> Нарушение этого правила = непроверенный продукт в репозитории = сломанная экосистема пользователя.

## Скилы (моя «память» о доступных возможностях)

### Установленные локально (`~/.claude/skills/`)
- **session-start-hook** — настройка SessionStart-хуков для Claude Code в вебе (чтобы тесты/линтеры
  работали в веб-сессиях). Главный кандидат для этого репо.

### Доступные через harness
**Конфигурация и автоматизация**
- **update-config** — настройка harness через `settings.json`: хуки (автоматические действия
  «каждый раз когда X»), permissions, env-переменные.
- **fewer-permission-prompts** — анализ транскриптов и автодобавление allowlist в
  `.claude/settings.json`, чтобы меньше спрашивать разрешений.
- **keybindings-help** — кастомизация горячих клавиш.
- **loop** — запуск задачи/команды по расписанию (например `/loop 5m ...`).

**Качество кода**
- **code-review** — ревью текущего диффа на баги и упрощения (уровни low→max; `--comment`, `--fix`).
- **simplify** — только упрощения/переиспользование/эффективность (без поиска багов).
- **security-review** — security-ревью изменений на ветке.
- **review** — ревью GitHub PR (для рабочего диффа — `/code-review`).
- **verify** / **run** — запуск приложения, чтобы убедиться, что изменение реально работает.

**Документация и инициализация**
- **init** — создать/обновить `CLAUDE.md` документацией по кодовой базе.
- **claude-api** / **claude-code-guide** — справка по Claude API/SDK и по самому Claude Code.

### Подключённые MCP-серверы
- **github** — все операции с GitHub (PR, issues, CI, поиск кода). `gh` CLI недоступен — только эти тулы.
- **Gmail** — поиск/чтение тредов, черновики, метки.
- **Google Drive** — поиск/чтение/создание файлов.

## Рабочие договорённости
- Ветка разработки: `claude/skills-memory-improvement-vo4cyy`. На неё коммитить и пушить.
- PR создавать только по явной просьбе.
- **После слияния PR — удалять его смёрженную head-ветку**, чтобы ветки не копились.
  Исключение: **не удалять активную ветку разработки** (см. строку выше), если на ней
  продолжается работа. Удаление: `git push origin --delete <branch>`. Полная автоматика
  без моего участия — галочка **Settings → General → «Automatically delete head branches»**
  в репозитории (включается владельцем вручную; MCP-тула для неё нет).
- Сетевой доступ — через прокси (см. `/root/.ccr/README.md` при TLS/403/407).

## ⚠️ Известные ограничения окружения
- **Запись в репозиторий (push) может быть недоступна.** На 2026-06-27 и `git push`
  (git-релей `127.0.0.1`), и GitHub MCP (`push_files`) возвращали `403` — у сессии
  не было write-доступа к `kkirilla-collab/markitdown` (чтение работает). Если снова
  ловлю 403 на push — это политика доступа, **не повторять бесконечно**: коммитить
  локально и сообщить пользователю, что нужно выдать права (Contents: Read and write
  для GitHub App). Не тратить попытки на обход.
  - **`403 Resource not accessible by integration` (git push и GitHub MCP) = у Claude
    GitHub App нет доступа на запись к этому репо, а не проблема токена/сессии.** Рестарт
    НЕ помогает (старая сессия токен не обновляет). На 2026-06-28 решилось так: пользователь
    дал приложению Claude **доступ именно к `markitdown`** (GitHub → Settings → Applications
    → Installed GitHub Apps → Claude → Configure → Repository access), после чего **в новой
    сессии** push сразу прошёл. Одобрение прав/рестарт сами по себе не помогали — ключевое
    было дать App доступ к репозиторию + начать новую сессию. Альтернатива: `/web-setup`
    (синхронизация личного `gh`-токена) — пуш идёт под пользователем, минуя App.
- **Локальная подпись коммитов невозможна.** Git настроен на ssh-подпись
  (`commit.gpgsign=true`, `gpg.format=ssh`, `user.signingkey=/home/claude/.ssh/commit_signing_key.pub`),
  но этот файл **пустой (0 байт)** и принадлежит юзеру `claude`, а сессия идёт под `root`
  (`HOME=/root`); приватного ключа нет. Поэтому `git commit -S` не создаёт подпись
  (`%G?` = `N`) и verify падает на `gpg.ssh.allowedSignersFile`. Верификация
  проставляется на стороне git-релея **при push** — то есть лечится тем же грантом
  write-прав. **Не тратить шаги на локальную подпись/reset-author ради подписи:**
  достаточно убедиться, что committer email = `noreply@anthropic.com`.

## Окружение для разработки (как поднять)
- `markitdown` ставится editable: `pip install -e "packages/markitdown[all]"`.
- Инструменты: `pytest` (тесты), `black` (форматирование, см. `.pre-commit-config.yaml`).
- Это автоматизировано в `.claude/hooks/session-start.sh` (SessionStart-хук для веб-сессий).
- **`tests/test_module_misc.py::test_speech_transcription` в этом окружении принципиально
  красный** — даже с установленным `ffmpeg` (хук его ставит) `AudioConverter` ходит в
  онлайн-сервис распознавания речи, а прокси возвращает `Forbidden`. Это ограничение сети,
  не баг кода — тест сразу пропускать, не диагностировать заново. Аналогично
  `test_markitdown_remote` требует внешней сети.

## Знания об экономии токенов Claude API

- **Оценка токенов:** 3–4 символа/токен — универсальная метрика для русского и английского.
  «0.75 слова/токен» работает только для английского.
- **Prompt Caching:** cache write стоит на 25% _дороже_ обычного input-токена;
  cache read — на 90% _дешевле_. Кэширование выгодно только если одинаковый префикс
  повторяется несколько раз за 5 минут (TTL кэша). При редких запросах кэш увеличивает счёт.
  В промпте статический текст всегда _перед_ динамическим — иначе кэш не попадает в цель.
- **Batch API:** для фоновых/асинхронных задач (без мгновенного ответа) — скидка 50% на все
  токены (input + output). Must-have для массовой обработки.
- **markitdown делает одноразовые LLM-запросы** (не диалоги) — управление историей, суммаризация
  контекста и каскад моделей к этой библиотеке не применимы.
