# CLAUDE.md — память проекта markitdown

Этот файл — постоянная память Claude для этого репозитория. Читается в начале каждой сессии.

## О проекте

`markitdown` — инструмент для конвертации файлов и офисных документов в Markdown.
Монорепо, пакеты в `packages/`:

| Пакет | Назначение |
|---|---|
| `markitdown` | Основная библиотека и CLI конвертации в Markdown |
| `markitdown-mcp` | MCP-сервер поверх markitdown |
| `markitdown-ocr` | OCR-расширение |
| `markitdown-sample-plugin` | Пример плагина (шаблон для расширений) |

Python-проект. Тесты/линтеры запускаются на уровне каждого пакета (см. `pyproject.toml` внутри пакетов).

## Скилы (моя «память» о доступных возможностях)

### Установленные локально (`~/.claude/skills/`)
- **session-start-hook** — настройка SessionStart-хуков для Claude Code в вебе (чтобы тесты/линтеры
  работали в веб-сессиях). Главный кандидат для этого репо.

### Доступные через harness
**Конфигурация и автоматизация**
- **update-config** — настройка harness через `settings.json`: хуки (автоматические действия
  «каждый раз когда X»), permissions, env-переменные.
- **fewer-permission-prompts** — анализ транскриптов и автодобавление allowlist в
  `.claude/settings.json`, чтобы меньше спрашивать разрешений.
- **keybindings-help** — кастомизация горячих клавиш.
- **loop** — запуск задачи/команды по расписанию (например `/loop 5m ...`).

**Качество кода**
- **code-review** — ревью текущего диффа на баги и упрощения (уровни low→max; `--comment`, `--fix`).
- **simplify** — только упрощения/переиспользование/эффективность (без поиска багов).
- **security-review** — security-ревью изменений на ветке.
- **review** — ревью GitHub PR (для рабочего диффа — `/code-review`).
- **verify** / **run** — запуск приложения, чтобы убедиться, что изменение реально работает.

**Документация и инициализация**
- **init** — создать/обновить `CLAUDE.md` документацией по кодовой базе.
- **claude-api** / **claude-code-guide** — справка по Claude API/SDK и по самому Claude Code.

### Подключённые MCP-серверы
- **github** — все операции с GitHub (PR, issues, CI, поиск кода). `gh` CLI недоступен — только эти тулы.
- **Gmail** — поиск/чтение тредов, черновики, метки.
- **Google Drive** — поиск/чтение/создание файлов.
